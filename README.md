# Курсовой проект: аренда торговых площадей, вариант 26

Проект сделан по структуре примера с сохранением двух отдельных приложений:

- `server/` — TCP-сервер Qt/C++, который подключается к PostgreSQL через `QSqlDatabase` и выполняет запросы через `QSqlQuery`.
- `client/` — desktop-клиент Qt Widgets, который подключается к серверу через `QTcpSocket`.
- `database/` — SQL-скрипты физической модели базы данных.

## Что реализовано

- Авторизация пользователей.
- Роли: `Администратор` и `Клиент`.
- Админ-панель с CRUD для основных таблиц.
- Просмотр и учет клиентов, торговых точек, договоров аренды, арендуемых точек и платежей.
- Быстрое оформление договора аренды администратором.
- Быстрое добавление платежа администратором.
- Кабинет клиента: свободные площади на выбранный период, свои договоры, свои платежи, регистрационные данные.
- Изменение пароля с проверкой сложности регулярным выражением.
- SQL-запросы параметрические: `prepare()` + `bindValue()`.
- На стороне клиента есть классы-модели: `User`, `Client`, `RetailSpace`, `ContractInfo`, `PaymentInfo`.

## Порядок запуска

1. В PostgreSQL создать базу данных `rental_spaces_db`.
2. Выполнить `database/create_tables.sql`.
3. Выполнить `database/update_password_hashes.sql`.
4. Открыть `server/server.pro` в Qt Creator и запустить сервер.
5. Открыть `client/client.pro` в Qt Creator и запустить клиент.

## Настройки подключения к БД

В файле `server/Database/databasemanager.cpp` указаны настройки:

- База данных: `rental_spaces_db`
- Пользователь: `postgres`
- Пароль: `Taras156`

Если пароль PostgreSQL другой, поменяйте константу `PG_PASSWORD`.

Сервер сначала пробует подключиться через `QPSQL`, затем через `QODBC`. Если появляется ошибка `Driver not loaded`, смотрите файл `database/odbc_setup.txt`.
